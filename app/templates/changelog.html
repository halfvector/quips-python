Changelog

2015-12-05

Dark-theme with unsplash.com bg - because I often work on this late at night.

More mobile friendly design.

Stopped trying to get audio-recording to work well on Android 4.x after burneing many weekends and nights. The audio glitches even when recording pure PCM, a problem at the Web Audio level, nothing I can do about it.

Found a fun workaround mobile chrome's inability to play Web Audio recorded wave files: run the generated blobs through an ajax request, making the blob disk-backed locally, now the local blob can be passed into an <audio> player.

Focusing on making the mobile listening experience great.

2015-10-04

Slight facelift, using a new flat style. Added a few animations and this public changelog! :)

2015-09-26

Designed a logo and created a pretty landing-page with twitter-login.

Added Sentry for Javascript error collection and Heap Analytics for creating ad-hoc analytics.

2015-09-20

Setup two new servers on Digital Oceans with Route 53 routing and an SSL certificate for production. Having an SSL certificate means the site can be accessed via HTTPS which allows browsers to cache the Microphone Access permissions, which means you don't have to click "allow" every time you want to make a recording!

Fixed up Python Fabric deployment script to work in new staging + production environments. And added MongoDB backup/restore support.

Updated Python dependencies, they were over a year old, and fixed code that broke as a result. Mostly around changes to MongoEngine Python lib.

2015-09-05

Fixed project to work on OSX and without the NGINX dependency. I can now run it all in python, including the static file hosting. The production servers use NGINX for better performance.

2014-03-29

Request Media Streaming permission from browser on recording-page load. This makes the microphone available the instant you hit the record button. No need to hit record button and then deal with browser's security popups asking for permission to access microphone.

Removed countdown clock untill recording begins, the "3-2-1 go" wasn't that helpful.

2014-03-27

Fixed bug in tracking where you paused in the playback of a recording. Now you should be able to resume playback exactly where you left off. :)
